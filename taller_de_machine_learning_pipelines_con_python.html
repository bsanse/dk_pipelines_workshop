Introducción¶

Los pipelines de machine learning se utilizan para automatizar los flujos de trabajo. Funcionan de tal manera que que se atomiza y encapsula cada uno de los procesos de tal manera que la salida de uno de los pasos es consumida como entrada del siguiente paso. Un ejemplo clsico de un pipeline de Machine Learning sera el siguiente:

En este taller vamos a ver un ejemplo de como utilizar los Pipelines de scikit-learn para el preprocesado, ingeniera de variables y entrenamiento de nuestros modelos de machine learning.

Las principales ventajas de los pipelines son las siguientes:

  • El flujo de trabajo es mucho ms fcil de leer y de entender
  • Fcil utilizacin de transformadores pblicos
  • Fcil reutilizacin de transformadores propios
  • Al tener el cdigo atomizado es fcil hacer pruebas cambiando una pieza por otra
  • Facilidad de productivizacin

Iteración 1: Nuestro primer Pipeline¶

A continuacin vamos a crear nuestro primer pipeline. En esta primera aproximacin slo utilizaremos las variables numricas.

Para crear nuestros pipeline primero definimos cada uno de los objetos del mismo:

  1. Imputar nulos mediante SimpleImputer
  2. Reescalar los datos con StandardScaler
  3. Regresin lineal mediante LinearRegression
In [7]:
num_imputer = SimpleImputer(strategy='median')
scaler = StandardScaler()
lr = LinearRegression()

Creamos una lista de tuplas en las que cada tupla corresponde a un par de ('nombre del paso', objeto).

In [8]:
steps = [
    ('numeric_imputer', num_imputer),
    ('scaler', scaler),
    ('regressor', lr)
]

Para crear el pipeline slo tenemos que utilizar Pipeline de scikit-learn y pasarle la lista de tuplas:

In [9]:
pipeline_first = Pipeline(steps)

Iteración 4: Pipeline Avanzado con Target Encoding¶

Vamos a aadir un encoder ms complejo para aquellas variables categricas de dimensionalidad alta. Para ello vamos a utilizar una variante de 'target encoder' llmada LeaveOneOutEncoder. Este tipo de encoders son muy potentes porque utilizan el target para codificar las variables categoricas capturando as correlaciones que puedan existir. Sin embargo, tienen un riesgo muy alto de sobreajustar, ya que se ha utilizado la informacin del target en el proceso. No es objeto de este taller, pero quien est ms intersado puede obtener ms informacin sobre 'target encoding' aqu: https://brendanhasz.github.io/2019/03/04/target-encoding

Nota friki: Target Encoding se utiliza mucho en las competiciones de Kaggle.

Nuestro pipeline quedara tal que as:

¿Cómo sabemos que valor es el correcto?¶

Aqu empezamos a ver el verdadero valor de los pipelines de scikit-learn. Como vamos a todo el proceso por validacin cruzada, podemos hacer una bsqueda de los hiperparmetros de los transformadores de la misma manera que lo hacemos habitualmente con los estimadores.

Si hubiese realizado el encoding sin utilizar pipelines, antes de aplicar validacin cruzada estara podra 'comer' todo el overfitting en el proceso de validacin y llevarme una sorpresa desagradable al realizar el test.

Iteración 5: Pipeline Avanzado con Transformadores Propios¶

Podemos incluir nuestros propios transformadores en los pipelines. Para ello tenemos que definir una clase con los mtodos fit y transform. Si utilizamos BaseEstimator de sckit-learn la aadira los mtodos para aadir y extraer parmetros. Si utilizamos TransformerMixin le aadir el mtodo fit_transform de manera automtica.

Vamos a realizar un cluster espacial para aadir la misma etiqueta a aquellas viviendas que se encuentran cerca. Esto puede aportarnos un valor ms objetivo que los barrios en los que estn las viviendas.

Definimos la clase utilizando KMeans:

In [30]:
class KMeansLatLon(BaseEstimator, TransformerMixin):
    def __init__(self, columns, n_clusters=10):
        self.columns = columns
        self.n_clusters = n_clusters
        self.model = KMeans(n_clusters=self.n_clusters)

    def fit(self, X, y=None):
        self.model.fit(X[self.columns])
        return self

    def transform(self, X, y=None):
        data = X.copy()
        pred = self.model.predict(data[self.columns])
        data['LatLonCluster'] = pred
        return data

Productivización¶

Dado que el pipeline incluye tanto el preprocesado como el estimador, al guardar este conseguimos encapsular en un slo fichero todo el proceso. Esto facilita muchsimo la puesta en produccin posterior de los modelos (sea medaiante API o meadiante proceso batch) porque bastara con cargar el 'pikle' que hemos generado, en lugar de ir adaptando todo el cdigo al entorno productivo:
